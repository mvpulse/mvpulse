# MVPulse Demo Script

## Live Demonstration Guide

**Duration**: 5-7 minutes
**Network**: Movement Testnet

---

## Pre-Demo Checklist

- [ ] Browser with MetaMask/Petra wallet installed
- [ ] Testnet wallet with MOVE tokens
- [ ] App running at https://mvpulse.vercel.app (or localhost)
- [ ] Two browser windows/tabs ready (Creator + Participant view)
- [ ] Movement testnet is operational (check RPC status)

---

## Demo Flow

### Part 1: Introduction (30 seconds)

**Say**: "MVPulse is a decentralized polling platform on Movement Network where users earn real token rewards for participating in polls. Let me show you how it works."

**Action**: Show the landing page

---

### Part 2: Wallet Connection (30 seconds)

**Say**: "Users can connect with traditional Web3 wallets like Petra, or use our Privy integration for social login - perfect for Web2 users entering Web3."

**Action**:
1. Click "Connect Wallet"
2. Show wallet options (Petra, Nightly, etc.)
3. Connect with Privy using email/Google
4. Point out the auto-funded testnet wallet

---

### Part 3: Creator Flow - Creating a Poll (1.5 minutes)

**Say**: "Let's create a poll as a project creator."

**Action**:
1. Navigate to **Creator** > **Create Poll**
2. Fill in:
   - Title: "What's your favorite blockchain feature?"
   - Options: "Speed", "Low Fees", "Security", "Developer Experience"
   - Category: Technology
   - Duration: 1 hour
3. Configure rewards:
   - Token: PULSE
   - Type: Equal Split
   - Total: 100 PULSE
4. Show platform fee (2%)
5. Click "Create Poll"
6. Confirm transaction

**Say**: "The poll is now live on-chain. The reward pool is locked in the smart contract until distribution."

---

### Part 4: Participant Flow - Voting (1 minute)

**Say**: "Now let's switch to a participant's perspective."

**Action**:
1. Open incognito/different browser
2. Connect a different wallet
3. Navigate to **Projects**
4. Find the poll we just created
5. Click on it to view details
6. Select an option and vote
7. Confirm transaction

**Say**: "The vote is recorded on-chain. The participant is now eligible for rewards when the poll closes."

---

### Part 5: Questionnaires (1 minute)

**Say**: "MVPulse also supports questionnaires - bundled surveys with shared reward pools."

**Action**:
1. Navigate to **Questionnaires**
2. Show an existing questionnaire
3. Highlight:
   - Multiple polls in sequence
   - Shared pool calculation
   - Bulk voting (one transaction for all)
   - Progress tracking

**Say**: "Participants complete all polls and claim their share of the reward pool."

---

### Part 6: Staking & Tiers (45 seconds)

**Say**: "Users can stake PULSE to unlock higher tiers with more daily votes."

**Action**:
1. Navigate to **Staking**
2. Show tier requirements:
   - Bronze: 0 PULSE = 3 votes/day
   - Platinum: 100,000 PULSE = 12 votes/day
3. Show staking interface with lock periods
4. (Optional) Stake a small amount

**Say**: "Longer lock periods may offer additional benefits in the future."

---

### Part 7: Referral System (30 seconds)

**Say**: "Our referral system incentivizes user growth."

**Action**:
1. Navigate to **Participant** > **Referrals**
2. Show referral code and shareable link
3. Show milestone system (First Vote, 10 Votes, etc.)
4. Show leaderboard

---

### Part 8: Project Organization (30 seconds)

**Say**: "Creators can organize their polls into projects with team collaboration."

**Action**:
1. Navigate to **Creator** > **Projects**
2. Show a project with multiple polls
3. Highlight:
   - Collaborator roles (Admin, Editor, Viewer)
   - Analytics dashboard
   - AI-powered insights

---

### Part 9: Token Swap (30 seconds)

**Say**: "Users can swap between PULSE and USDC using our built-in AMM."

**Action**:
1. Navigate to **Swap**
2. Show swap interface
3. Enter an amount
4. Show price quote and slippage settings

---

### Part 10: Wrap-Up (30 seconds)

**Say**: "To summarize, MVPulse offers:
- Token-rewarded polling with MOVE, PULSE, or USDC
- Questionnaire bundles with shared rewards
- Staking for tier benefits
- Referral rewards
- Project collaboration
- All built on Movement Network's fast, low-cost infrastructure."

**Action**: Return to landing page

---

## Backup Plans

### If testnet is slow:
- Have pre-recorded video clips ready
- Show already-created polls instead of creating new ones

### If wallet connection fails:
- Use a different wallet (Petra vs Nightly)
- Show Privy social login as alternative

### If transaction fails:
- Explain this is testnet and occasional issues occur
- Show the transaction that would be submitted
- Move to next demo section

---

## Key Talking Points

1. **On-chain transparency** - All votes and rewards are verifiable
2. **Multi-token flexibility** - Support for native, platform, and stable tokens
3. **Web2-friendly onboarding** - Privy social login + gas sponsorship
4. **Gamification** - Tiers, quests, seasons, referrals drive engagement
5. **Movement advantage** - Fast finality, low fees, Aptos Move security

## Questions to Anticipate

**Q: How do you prevent sybil attacks?**
A: Tier system with staking requirements, daily vote limits, and planned integration with on-chain identity solutions.

**Q: What's the revenue model?**
A: 2% platform fee on reward distributions, premium features for creators.

**Q: Why Movement Network?**
A: High TPS, low latency, Move language security, growing ecosystem.

**Q: What about mainnet?**
A: Contracts are testnet-ready, mainnet deployment planned post-hackathon.
